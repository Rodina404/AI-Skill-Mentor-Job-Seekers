1. Executive Summary

The CV Matching System is an intelligent platform that automatically matches job candidates with job descriptions using advanced machine learning and natural language processing techniques. The system combines semantic analysis with rule-based matching to provide accurate, explainable candidate-job fit assessments.

🎯 Primary Objective

Automate and optimize the candidate selection process by using AI-powered semantic matching, hybrid parsing, and multi-factor scoring to identify the best candidates for each job position.

2. System Overview

2.1 Architecture

The CV Matching System follows a modular, layered architecture designed for maintainability, scalability, and extensibility:

  • Input Layer: Accepts job descriptions and candidate data
  • Processing Layer: Parses, extracts, and vectorizes content
  • Search Layer: Performs semantic similarity matching
  • Scoring Layer: Applies multi-factor evaluation
  • Output Layer: Ranks and presents results with explanations

2.2 Core Components

Component Purpose Technology
Job Parser Extract skills, experience, tools from job descriptions Regex + Transformers
Embedder Convert text to vector representations HuggingFace Transformers
Vector Store Store and index embeddings for fast retrieval FAISS
Matcher Find semantically similar candidates Semantic Search (Cosine Similarity)
Scorer Calculate match scores with multiple factors Python + FuzzyWuzzy
Web Interface User-facing application and API Flask + REST API

3. Pipeline Flow

3.1 Five-Stage Processing Pipeline

Stage 1: Input Data Ingestion

Input: Job description (text) + Candidate database (JSON/CSV)

Process: Load and validate data

Output: Structured job posting and candidate list

Stage 2: Parsing & Requirement Extraction

Input: Raw job description

Process: Hybrid parsing using regex patterns and transformer models

Output: Extracted skills, required experience, tools list

Details:

  • Regex Phase: Fast extraction of common patterns (2-5ms)
  • Transformer Phase: Context-aware extraction for complex requirements (50-200ms)
  • Negation Handling: Prevents false positives from negated mentions
  • Synonym Normalization: Converts abbreviations (DL → Deep Learning)

Stage 3: Embedding & Vectorization

Input: Job description + Formatted candidate texts

Process: Convert to vector embeddings

Output: 384-dimensional vectors stored in FAISS index

Details:

  • Model: sentence-transformers/all-MiniLM-L6-v2
  • Embedding dimension: 384
  • Processing speed: ~5000 candidates/second on CPU
  • Cache: Persistent local storage

Stage 4: Semantic Search & Matching

Input: Job description vector + Candidate vectors

Process: Find k most similar candidates using cosine similarity

Output: Ranked list of candidates with similarity scores

Details:

  • Similarity metric: Cosine Similarity
  • Search complexity: O(log n) with FAISS index
  • Returns: All candidates with scores 0.0-1.0

Stage 5: Multi-Factor Scoring & Ranking

Input: Candidate + Parsed requirements + Similarity scores

Process: Apply weighted scoring across multiple factors

Output: Final match percentage (0-100%) with detailed breakdown

Scoring Formula:

final_score = (
    semantic_score × 0.40 +
    skill_score × 0.35 +
    tools_score × 0.15 +
    experience_score × 0.10
) × 100

5. Techniques & Algorithms

5.1 Natural Language Processing

5.1.1 Hybrid Job Parsing

Why Hybrid? Combines speed (regex) with accuracy (transformers)

  • Regex Phase (Fast):
    • Pattern matching for common skill names
    • Experience extraction with patterns like "X+ years"
    • Tool/framework identification
    • Processing time: 2-5ms per document
  • Transformer Phase (Accurate):
    • Context-aware requirement understanding
    • Handles complex/ambiguous descriptions
    • Fallback when regex patterns don't match
    • Processing time: 50-200ms per document
  • Negation Handling:
    • Detects negative contexts: "NOT required", "without", "don't need"
    • Prevents false positives from negated mentions
    • Window-based backward scanning (60 characters)
  • Synonym Normalization:
    • Maps abbreviations to full names: DL → Deep Learning
    • Normalizes skill names: "AI" → "Artificial Intelligence"
    • Consistent output format

5.1.2 Text Embedding

Model: sentence-transformers/all-MiniLM-L6-v2

  • Type: Sentence-level transformer embeddings
  • Dimension: 384-dimensional vectors
  • Training: Pre-trained on 215M sentence pairs
  • Speed: ~5000 sentences per second (CPU)
  • Advantage: Lightweight yet effective for semantic understanding

5.2 Machine Learning Techniques

5.2.1 Vector Similarity Search

Algorithm: FAISS with Cosine Similarity

  • Index Type: Flat L2 (Euclidean distance converted from cosine)
  • Similarity Metric: Cosine Similarity: cos(θ) = A·B / (|A||B|)
  • Range: 0.0 (completely different) to 1.0 (identical)
  • Complexity: O(n) for exact search, O(log n) with advanced indexing
  • Scalability: Can handle 1M+ vectors on single machine

5.2.2 Fuzzy String Matching

Algorithm: Levenshtein Distance (FuzzyWuzzy)

  • Purpose: Match skill names despite typos/variations
  • Method: Edit distance calculation (min operations to transform string A to B)
  • Threshold: 80% match confidence for skills
  • Usage: Compare candidate skills against job requirements

5.2.3 Multi-Factor Weighted Scoring

Approach: Normalized weighted sum of evaluation factors

Scoring Formula:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Semantic Similarity (40% weight)
   - Vector cosine similarity between job & candidate
   - Range: 0-1, normalized to 0-100

2. Skill Matching (35% weight)
   - Matching skills / Total required skills
   - Fuzzy matching with 80% threshold
   - Formula: len(matching) / max(required, 1)

3. Tools/Technologies Matching (15% weight)
   - Matching tools / Candidate's tools
   - Range: 0-1

4. Experience Level (10% weight)
   - Candidate experience / Required experience
   - Capped at 1.0 (max score)
   - Formula: min(candidate_exp / required_exp, 1.0)

Final Score = (Σ(factor × weight)) × 100
Range: 0-100%

7. Key Features

7.1 System Advantages

  • 🎯 Hybrid Parsing: Combines speed (regex) with accuracy (transformers) for robust requirement extraction
  • 📊 Semantic Intelligence: Goes beyond keyword matching to understand true job-candidate fit using embeddings
  • ⚡ Fast & Scalable: FAISS enables sub-second similarity search across thousands of candidates
  • 🔒 Self-Contained: Runs locally with no external API dependencies - complete privacy control
  • 🎓 Explainable Results: Detailed breakdown shows matching/missing skills and individual factor scores
  • ⚙️ Flexible Configuration: Easily tune weights, thresholds, and parser settings for specific needs
  • 🚀 Production Ready: Flask web app with REST API for easy integration
  • 📈 Continuous Improvement: Built-in logging and benchmarking for performance monitoring

7.2 Extensibility

  • Modular Design: Each component can be independently updated/replaced
  • Pluggable Embeddings: Easy to swap embedding models
  • Custom Parsers: Support for domain-specific parsing rules

  • Configurable Weights: Adjust scoring weights per organization needs
